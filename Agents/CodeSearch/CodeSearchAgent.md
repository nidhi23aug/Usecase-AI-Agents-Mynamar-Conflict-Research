## Relevant Geospatial / Foundation Model Repositories

Below are repositories that provide **building blocks for detecting spatiotemporal cropland abandonment and recultivation** using satellite data, foundation models, and geospatial ML pipelines.

---

### 1️⃣ NASA-IMPACT / Prithvi-EO-2.0  
**Repository:** https://github.com/NASA-IMPACT/Prithvi-EO-2.0  

**Why it is relevant**

- Directly targets **multi-temporal Earth observation foundation modeling**.  
- Implements the **Prithvi-EO-2.0 geospatial foundation model**.  
- Includes examples for **fine-tuning and inference on satellite imagery**.  
- Useful for **representation learning from large satellite time-series datasets**.

---

### 2️⃣ NASA-IMPACT / HLS-Foundation-OS  
**Repository:** https://github.com/NASA-IMPACT/hls-foundation-os  

**Why it is relevant**

- Demonstrates **end-to-end fine-tuning workflows** for the **Prithvi foundation model**.  
- Uses **Harmonized Landsat Sentinel (HLS)** satellite datasets.  
- Provides segmentation pipelines useful for **land-use and cropland monitoring tasks**.  
- Helpful for implementing **operational foundation model inference pipelines**.

---

### 3️⃣ Microsoft / TorchGeo  
**Repository:** https://github.com/microsoft/torchgeo  

**Why it is relevant**

- A widely used **geospatial machine learning library for PyTorch**.  
- Provides tools for:
  - geospatial dataset loading  
  - sampling strategies  
  - transformations  
  - pretrained model utilities  
- Useful for building **custom remote sensing ML workflows**.

---

### 4️⃣ NASA Harvest / CropHarvest  
**Repository:** https://github.com/nasaharvest/cropharvest  

**Why it is relevant**

- Agriculture-focused **remote sensing dataset and ML benchmark**.  
- Includes tools for:
  - crop classification  
  - agricultural monitoring  
  - Earth Engine export utilities  
- Useful for **training models on agricultural land-use signals**.

---

### 5️⃣ NASA NCCS HPDA / iLab Pangaea Bench  
**Repository:** https://github.com/nasa-nccs-hpda/ilab-pangaea-bench  

**Why it is relevant**

- Benchmark framework for **geospatial foundation models** including **Prithvi**.  
- Includes evaluation tasks such as:
  - change detection  
  - multi-temporal analysis  
  - geospatial classification  
- Useful for **evaluating EO foundation model performance**.

---

### 6️⃣ SAR Change Detection  
**Repository:** https://github.com/janne-alatalo/sar-change-detection  

**Why it is relevant**

- Deep learning framework for **change detection using SAR imagery**.  
- Particularly useful in **cloud-prone regions** where optical imagery is limited.  
- SAR methods can help detect **land-use change during monsoon seasons in Myanmar**.

---

## Notes

- No repository in the **NASA-verified corpus** appears to be specifically built for  
  **“cropland abandonment and recultivation in Myanmar after 2021.”**

- The repositories listed above are **enabling components**, including:
  - Earth-observation foundation models
  - training pipelines
  - agricultural datasets
  - change-detection frameworks

These can be combined to build a **custom workflow for detecting agricultural abandonment using satellite time series data**.
